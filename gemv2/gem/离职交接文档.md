## 一、项目概述

GEM v2 是 GEM 框架的重构版本，在保持 Hydra 配置驱动核心理念的基础上，做了以下关键改进:

1. **引入 `core/` 层**: 纯数据类型独立于业务逻辑，消除循环依赖
2. **组合替代继承**: `ModelPipeline` 取代 `BaseMethod`，组件注入而非子类化
3. **Ensemble Pipeline**: 原生支持多模型组合 + 可插拔融合策略
4. **模块拆分**: GBDT/sklearn 按框架和模型拆分为独立子包
5. **Transform 提升**: 从 `method/base/` 提升至顶层包，降低耦合

仓库根目录 `/home/user165/workspace/quant/gemv2/` 还包含独立子项目: `xgboost/`, `spectral_cluster/`, `deepcluster/`, `tangles_ens/`, `train_code/`。

## 三、核心数据流

```
YAML 配置 → Hydra 组合 → bootstrap.py → RuntimeBundle → ExperimentManager.run()
  │
  ├─ [1] SplitGenerator.generate()
  │       → SplitPlan (splitspec_list, date_start, date_end)
  │
  ├─ [2] DataModule.build_dataset(date_start, date_end)
  │       → FeatherReader.read_sources()                # polars DataFrames
  │       → SourcePipeline.fit_transform() per source   # 单源预处理
  │       → CrossSourcePipeline.fit_transform()         # AlignPreprocessor (join)
  │       → FeatureAssembler.assemble()                 # → GlobalDataset (numpy f32)
  │
  ├─ [3] _build_tasks() → List[SplitTask]
  │
  ├─ [4] RollingState() 初始化
  │
  ├─ [5] TaskDAG → Executor → 每个窗口:
  │       SplitRunner.run():
  │         GlobalDataset.take(indices) → SplitBundle (train/val/test)
  │         PipelineFactory.build()     → ModelPipeline 或 EnsemblePipeline
  │         Pipeline.run():
  │           ① TransformPipeline.fit_transform_views() → ProcessedBundle
  │           ② BaseTuner.tune()      (n_trials > 0 时)
  │           ③ TrainConfig + best_params
  │           ④ BaseTrainer.fit()     → FitResult
  │           ⑤ predict train/val/test
  │           ⑥ BaseEvaluator.evaluate() → Dict[str, EvalResult]
  │           ⑦ BaseImportanceExtractor.extract()
  │           ⑧ 保存 model.txt + feature_importance.csv
  │           ⑨ → RunOutput
  │         → SplitResult
  │       更新 RollingState (EMA)
  │
  ├─ [6] ReportGenerator.generate()
  │       → results_summary.csv, daily_metric_series.csv, config.json
  │
  └─ [7] run_visualization_pipeline() (若启用)
```

### EnsemblePipeline 流程 (v2 特有)

```
EnsemblePipeline.run():
  ├─ 对每个子 ModelPipeline:
  │    Pipeline_i.run() → RunOutput_i (含 predictions, metrics)
  ├─ BaseEnsembleStrategy.fit(val_predictions, val_y)
  ├─ strategy.combine(test_predictions) → 融合预测
  └─ evaluator.evaluate(融合预测) → 最终 EvalResult
```

## 四、关键数据结构 (core/ 层)

`core/` 层的类型只依赖 numpy 和 polars，无 gem 内部导入:

| 类 | 位置 | 说明 |
|----|------|------|
| `SplitSpec` | `core/data.py` | split_id + train/val/test 日期列表 (int YYYYMMDD) |
| `SplitPlan` | 同上 | SplitGenerator 输出: splitspec_list + date_start/end |
| `SourceSpec` | 同上 | 数据源描述: name, path, layout, date_col, code_col, pivot |
| `DatasetSpec` | 同上 | X/y/extra 源映射 + key_cols + group_col |
| `GlobalDataset` | 同上 | 全局数据: X_full (np.f32), y_full (np.f32), keys (pl.DataFrame), `_date_to_indices` |
| `SplitView` | 同上 | 单分区 X/y/keys |
| `SplitBundle` | 同上 | train/val/test 三元组 (v1 名: SplitViews) |
| `ProcessedBundle` | 同上 | Transform 后: 含 feature_mask, transform_state (v1 名: ProcessedViews) |
| `TrainConfig` | `core/training.py` | 训练超参 |
| `FitResult` | 同上 | model + evals_result + best_iteration |
| `EvalResult` | 同上 | 指标字典 + 日粒度序列 |
| `RunOutput` | 同上 | Pipeline 最终输出: metrics, importance, StateDelta |
| `StateDelta` | 同上 | 跨窗口传递: importance_vector, best_params, best_objective |
| `TransformContext` | 同上 | Transform 管道上下文 (rolling state 信息) |

依赖层次: `core/` ← `data/` ← `transforms/` ← `method/` ← `experiment/`

## 七、超参调优

v2 将调参后端从 `UnifiedTuner` 拆分为 `BaseTuner` ABC + 独立后端:

| 后端 | 位置 | 场景 |
|------|------|------|
| Optuna | `method/tuning/optuna.py` | 单机 (默认) |
| Ray Tune | `method/tuning/ray_tune.py` | 分布式 |
| NNI | `method/nni/tuner.py` | 神经架构搜索 |

关键特性:
- **Warm Start**: 从上一窗口最优参数初始化
- **Space Shrinking**: 缩小搜索范围到最优参数附近
- **并行 Trial**: `parallel_trials` 控制并发数

## 八、滚动状态 (RollingState)

与 v1 结构一致，跨时间窗口传递信息:

| State | 功能 | 下游消费者 |
|-------|------|-----------|
| `FeatureImportanceState` | 特征重要性 EMA | FeatureWeightTransform |
| `TuningState` | 保存最优超参 | BaseTuner warm start |
| `SampleWeightState` | 样本权重 | 训练时样本加权 |

执行模式 (`TaskDAG`):
- `none` — 无状态, 全并行
- `per_split` — 串行, 每窗口更新 (默认)
- `bucket` — 组内并行, 组间串行

EMA 平滑: `ema_alpha` (默认 0.3), 支持 topk 和 normalization

## 十一、与 v1 的关键差异

| 维度 | v1 (`gem/`) | v2 (`gemv2/gem/`) |
|------|-------------|-------------------|
| 数据类型 | 分散在 `data/data_dataclasses.py` + `method/method_dataclasses.py` | 统一到 `core/data.py` + `core/training.py`, 无内部依赖 |
| 核心流水线 | `BaseMethod` (继承式) | `ModelPipeline` (组合式) + `EnsemblePipeline` |
| 工厂 | `MethodFactory` | `PipelineFactory` (自动检测 ensemble 配置) |
| Transform | `method/base/transforms/` (嵌套) | `transforms/` (顶层包) + `spectral.py` |
| GBDT | 扁平文件 (`trainers.py` 含 3 个类) | 按框架拆分: `lightgbm/`, `xgboost/`, `catboost/` |
| sklearn | 单 Trainer 处理所有模型 | 按模型拆分: `ridge/`, `lasso/`, ... 各有 `config.py` |
| 调参 | `UnifiedTuner` (method/base/tuning/) | `BaseTuner` ABC + `method/tuning/` 后端 + NNI |
| Ensemble | 不支持 | `method/ensemble/` (ModelPipeline 组合 + 策略) |
| 报告 | `experiment/report.py` (单文件) | `experiment/report/` (子包, 含衍生指标计算) |
| 命名风格 | `data_module.py`, `base_trainer.py`, ... | `module.py`, `trainer.py`, ... (去前缀) |
| 配置路径 | `global_data_assembler/`, `multi_source_data_preprocessor_pipeline/` | `dataset_assembler/`, `cross_source_pipeline/` (更简洁) |
| 关键类名 | `GlobalStore`, `SplitViews`, `ProcessedViews`, `MethodOutput` | `GlobalDataset`, `SplitBundle`, `ProcessedBundle`, `RunOutput` |
| 子项目 | 无 | xgboost/, spectral_cluster/, deepcluster/, tangles_ens/ |

## 十二、设计约定

- Hydra `_target_` 驱动所有组件实例化, 纯 YAML 切换
- 只支持 Feather/IPC 格式 (polars `read_ipc`)
- 日期: int YYYYMMDD (配置), pl.Datetime("us") (内部)
- 多源特征命名: `"source__column"` 避免冲突
- GlobalDataset 全内存, numpy float32
- `core/` 层只依赖 numpy + polars, 保证类型可独立测试
- `PipelineFactory` 根据配置是否含 `ensemble` 键自动选择 `ModelPipeline` 或 `EnsemblePipeline`
