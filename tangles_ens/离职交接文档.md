>
> 最后更新: 2026-04-11

---

## 一、项目概述

Tangles Ensemble 是一个量化多模型集成框架，核心思路是利用 **Tangle 图论聚类** 对股票进行分群，在每个聚类内根据各基准模型的历史 IC（信息系数）动态分配权重，从而实现 **per-cluster IC-weighted ensemble**。相比简单等权平均，该方法能利用模型间的异质性，减少冗余、提升风险调整后的收益。

**核心价值**: 6 个基准模型之间相关性较高（0.64–0.81），直接等权平均会引入冗余。Tangle 聚类能识别股票的协议结构（agreement structure），在不同聚类中赋予不同模型不同的权重，使集成信号更具多样性和预测力。

## 三、核心架构与数据流

### 整体流程

```
config.py (路径/超参)
    ↓
data_loader.py
    load_data() → (date, code, bench1-6, label, can_trade)
    ↓
backtest.py / visualize.py
    对每个评估日期:
    ├── 1. 取当日截面数据 + 历史 lookback 窗口
    ├── 2. tangle_ensemble.ensemble_one_date():
    │      a. rank_normalize() → 截面百分位排名
    │      b. generate_cuts_from_scores() → 分位数二元切分
    │      c. run_tangle_clustering() → Tangle 树聚类 → cluster_labels
    │      d. compute_cluster_weights() → 每个聚类的 IC 加权权重
    │      e. 对每只股票: score = Σ w[cluster,j] × bench[j]
    ├── 3. 组合构建: 按 score 排序 → 前 500 只 (受流动性约束)
    └── 4. 计算日收益率 + Spearman IC
    ↓
结果聚合: IC / ICIR / top_return / stability + 季度分解
    ↓
visualize.py → plots/*.png
```

### 关键数据结构

| 数据 | 格式 | 说明 |
|------|------|------|
| 基准模型分数 | Feather (wide) → Polars long | bench1–bench6，6 个模型的截面打分 |
| 标签 (label) | Feather (wide) → Polars long | 日频股票收益率 |
| 流动性 (can_trade) | Feather (wide) → Polars long | 可交易金额约束 |
| 合并数据 | Polars DataFrame | (date, code, bench1-6, label, can_trade)，按 (date, code) 排序 |

### 4.3 tangle_ensemble.py（核心）

| 函数 | 说明 |
|------|------|
| `rank_normalize(scores)` | 截面百分位排名，NaN-safe，输出 ∈ [0, 1] |
| `generate_cuts_from_scores(scores, n_quantiles)` | 对每个 bench 生成分位数二元切分 (n_benches × n_quantiles 个切分) |
| `cut_cost_balance(n_samples, cut)` | 切分代价函数: 1 - min_side/total，惩罚不平衡切分 |
| `run_tangle_clustering(scores)` | **主聚类函数**: rank → cuts → tangle 计算 → 收缩树 → 剪枝 → 硬聚类标签 |
| `run_tangle_full(scores)` | 同上但返回中间对象 (用于可视化) |
| `compute_cluster_weights(hist_df, labels, codes, bench_cols)` | 每聚类内计算各 bench 的 Spearman IC → 权重 = max(IC, 0) 归一化 |
| `ensemble_one_date(date_df, hist_df, bench_cols)` | 单日集成: 聚类 → 权重 → 加权打分; 聚类失败时回退等权 |

**集成公式**:
```
stock i ∈ cluster c:
  score[i] = Σ(j=1..6) w[c,j] × bench[j][i]

其中 w[c,j] = max(IC(bench_j, label) in cluster c, 0) / Σ max(IC, 0)
```

## 五、Tangle 聚类算法详解

### 原理

Tangle 是一种图论分区方法，不同于 K-Means 需要预设聚类数。它通过以下步骤自动发现聚类:

1. **生成切分 (Cuts)**: 对每个 bench 在 4 个分位数阈值处生成二元切分 → 共 6×4 = 24 个候选切分
2. **过滤退化切分**: 两侧至少各有 ≥5 只股票
3. **计算切分代价**: 基于高斯核距离 (特征空间中的相似性) + 平衡性惩罚
4. **Tangle 计算**: 寻找一致性集合 — 满足 agreement 阈值 (5% 股票数) 的切分方向组合
5. **收缩树**: 将 Tangle 树简化为更紧凑的表示
6. **剪枝**: 限制树深度为 1，得到扁平聚类
7. **硬聚类**: 从软预测转为离散标签

### 依赖库

使用外部库 `tangles_src.tangles`（需在父目录或 Python 路径中）:
- `Cuts`: 切分数据结构
- `tangle_computation()`: 核心算法
- `ContractedTangleTree`: 收缩树
- `cost_functions`: 高斯核代价函数

## 六、评估指标

| 指标 | 计算方式 | 含义 |
|------|----------|------|
| **IC** | avg(Spearman(score_rank, return_rank)) | 预测力: 打分排名与实际收益排名的相关性 |
| **ICIR** | IC / std(IC) | 预测稳定性: 风险调整后的 IC |
| **top_return** | avg(daily P&L / capital) | 头部组合日均收益率 |
| **top_return_stability** | top_return / std(return) | 收益稳定性 (类 Sharpe) |
| **diversity_entropy** | 特征值分解归一化熵 | 集成多样性 ∈ [0, 1]，越高越多样 |

### 参考基线

- Bench 等权平均: diversity_entropy ≈ 0.527
- 个别 bench 相关性: 0.64–0.81 (高共线性)
- hhx/dynamic3 基线: IC=0.1099, RankIC=0.1145

## 八、设计决策与注意点

### 关键设计决策

| 决策 | 原因 |
|------|------|
| Tangle 聚类而非 K-Means | 不需预设聚类数，自动发现协议结构 |
| 剪枝深度 = 1 | 扁平聚类足以区分主要模式，深层聚类样本不足 |
| IC 加权 (非等权) | 不同聚类中各 bench 预测力不同，IC 加权更合理 |
| max(IC, 0) 截断 | 避免负 IC 模型贡献负权重 |
| 60 日 lookback | 平衡近期信息和统计显著性 |
| 聚类失败回退等权 | 保证鲁棒性，避免极端情况下无输出 |
| 日频换仓 | 与基准模型一致的评估频率 |
| 流动性约束贪心建仓 | 模拟真实交易限制 |

### 注意事项

1. **tangles_src 依赖**: 该库非标准 pip 包，需确保 `tangles_src/` 目录在项目父目录或 `sys.path` 中
2. **数据格式**: 仅支持 Feather (IPC) 格式，通过 `polars.read_ipc` 读取
3. **宽表约定**: bench/label/liquidity 文件均为 wide 格式 (行=日期, 列=股票代码)
4. **Inner Join 对齐**: 6 个 bench 做 inner join，只保留所有模型都有打分的股票
5. **NaN 处理**: rank_normalize 和 weight 计算均为 NaN-safe，但大量缺失会影响聚类质量
6. **MIN_CLUSTER_SIZE = 30**: 聚类内不足 30 只股票时回退等权，避免 IC 估计不稳定
7. **评估首月跳过**: 20230101 加载数据但从 20230201 开始评估，首月作为预热

## 九、超参调优方向

| 参数 | 当前值 | 可探索范围 | 影响 |
|------|--------|------------|------|
| `AGREEMENT_RATIO` | 0.05 | 0.03–0.10 | 越小聚类越多，越大聚类越少 |
| `N_QUANTILES` | 4 | 3–6 | 越多切分越细，但增加计算量 |
| `LOOKBACK_DAYS` | 60 | 30–120 | 越长越稳定，越短越适应近期变化 |
| `PRUNE_DEPTH` | 1 | 0–2 | 越深聚类越多层次 |
| `MIN_CLUSTER_SIZE` | 30 | 20–50 | 越小允许更小聚类，越大更保守 |
| `MAX_STOCKS` | 500 | 200–800 | 组合集中度 vs 分散化 |
